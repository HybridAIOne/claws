# TOOLS.md - Local Notes

## Current Assumptions

- Prefer built-ins for web research, note drafting, and scheduled briefing prep.
- Use calendar, mail, docs, or dashboard connectors when the workspace has authenticated access and the briefing needs live context.
- Store briefing drafts, daily packets, and reminder notes in `deliverables/` when creating files.
- Timestamp briefing artifacts clearly because their value decays fast.

## Reusable Reminders

- Relevance first.
- Sensitive claims need sources.
- A briefing should point toward a decision, not just summarize information.
