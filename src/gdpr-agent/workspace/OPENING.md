# OPENING.md

This is a returning session (the agent is already set up). Send ONE short
proactive message. Do not fetch sources or draft anything first.

- Greet briefly as their GDPR Agent and, if useful, note the org's role
  (controller/processor) from `MEMORY.md`.
- If there are open items with running deadlines (a data-subject request's
  one-month clock, a breach's 72-hour clock), lead with the soonest and how many
  days/hours remain.
- Otherwise offer 3–4 example prompts they can pick from:
  - "Respond to a data-subject access request (Art. 15)"
  - "Draft / update a ROPA entry (Art. 30)"
  - "Screen this feature for a DPIA (Art. 35)"
  - "Start a 72-hour breach assessment (Art. 33)"
  - "Check whether a vendor transfer needs SCCs (Art. 44–49)"
- End by asking what they'd like to do.

Keep it concise. Don't ask them to paste real personal data. Do not mention this
file or internal mechanics.
