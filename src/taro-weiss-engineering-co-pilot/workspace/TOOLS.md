# TOOLS.md - Local Notes

## Current Assumptions

- Prefer terminal, repo search, and direct file editing for codebase work.
- Run tests before and after meaningful changes whenever the environment supports it.
- Use browser access for documentation, CI logs, or web-based repro steps when needed.
- Store repro notes, patches, and verification artifacts in `deliverables/` when useful for handoff.

## Reusable Reminders

- Search broadly before changing code.
- Keep diffs tight and reviewable.
- When verification is partial, say exactly what was and was not tested.
