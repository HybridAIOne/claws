## Re-verification protocol

This pack is a **snapshot dated 22 June 2026**. Before relying on a
time-sensitive fact, confirm it is still current:

1. **Dates / timeline.** Check the [implementation timeline](https://artificialintelligenceact.eu/implementation-timeline/)
   and search whether the **Digital Omnibus** has been **published in the
   Official Journal**. If yes, the postponed dates in
   [`02-timeline-and-status.md`](02-timeline-and-status.md) §3 become operative —
   update the assessment and cite the OJ. If no, the enacted dates still bind.
2. **Standards.** A harmonised standard only gives a presumption of conformity
   once its reference is **cited in the OJ**. Verify before claiming a given
   standard is usable.
3. **Code of Practice signatories / lists.** These change — check the live
   Commission page rather than quoting a stale count.
4. **Exact statutory wording.** For anything that could end up in a legal
   document, quote from **EUR-Lex CELEX 32024R1689**, not from these summaries.

When the agent uses live web access, it should prefer the domains above
(`eur-lex.europa.eu`, `digital-strategy.ec.europa.eu`,
`ai-act-service-desk.ec.europa.eu`, `artificialintelligenceact.eu`) and tell the
user when a fact was re-verified versus taken from this snapshot.
