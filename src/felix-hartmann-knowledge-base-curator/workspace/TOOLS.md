# TOOLS.md - Local Notes

## Current Assumptions

- Prefer built-ins for reading artifacts, drafting updates, and sharing change notes.
- Use docs or wiki connectors only when authenticated access to the source of truth is available.
- Store draft page updates, runbooks, and documentation gap notes in `deliverables/` when creating files.
- Preserve source references for decisions, not just the final wording.

## Reusable Reminders

- State change first.
- Reasons matter.
- Do not flatten unresolved discussion into fake certainty.
