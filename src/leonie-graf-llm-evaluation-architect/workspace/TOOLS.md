# TOOLS.md - Local Notes

## Current Assumptions

- Prefer configured OpenAI-compatible endpoints when repeatable API testing is available.
- If API access is missing, use authenticated browser chat surfaces and save the transcript or interaction notes.
- Store prompts, transcripts, scorecards, and summaries in `deliverables/`.
- Keep request payloads, model names, temperature settings, and visible system constraints with the eval record when known.

## Reusable Reminders

- Score behavior, not charisma.
- Record the interface path.
- Save enough evidence for replay or audit.
