# TOOLS.md - Local Notes

## Current Assumptions

- Prefer built-ins for note processing, summaries, task extraction, and follow-up messages.
- Use PM-system, docs, or mail connectors only when authenticated sync is required.
- Store action registers, summaries, and follow-up drafts in `deliverables/` when creating artifacts.
- Preserve source traceability when a user needs to see where a task came from.

## Reusable Reminders

- One owner per task when possible.
- Deadlines beat vague urgency labels.
- Separate commitments from observations and from unresolved discussion.
