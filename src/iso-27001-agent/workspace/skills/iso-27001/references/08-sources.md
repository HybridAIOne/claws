# Sources, Copyright & Re-verification

*Compiled 22 June 2026.*

## Copyright note (important)

ISO/IEC 27001 and ISO/IEC 27002 are **copyrighted standards** sold by ISO. This
reference pack:

- summarises the **structure and requirements** in original wording;
- lists control **reference numbers and short titles**, which are factual
  identifiers needed to reference the standard; and
- **paraphrases** the intent of each control.

It does **not** reproduce the normative text or ISO 27002's implementation
guidance. For audit-grade wording, work from a **licensed copy** purchased from
ISO or a national standards body. The agent must not paste long verbatim extracts
of the standard even if a user asks.

## Re-verification protocol

This pack is a **snapshot dated 22 June 2026**. Before relying on a time-sensitive
fact:

1. **Edition / amendment.** Confirm **27001:2022 + Amd 1:2024** is still current
   and watch for any later amendment or a new edition.
2. **Transition.** The 2013→2022 transition ended **31 Oct 2025**; treat 2013 as
   withdrawn. If a user cites a 2013 certificate, flag it.
3. **No prescribed numbers.** ISO does **not** set specific log-retention,
   patch-SLA, MFA-factor, or review-cadence values — these are org-defined risk
   decisions. Treat any specific number from a vendor blog as suggested practice,
   not the standard.
4. **Exact wording.** For anything audit-grade, quote from the **licensed
   standard**, not these summaries.

When the agent has live web access, prefer **iso.org** and accredited bodies
(UKAS, ANAB, IAF) and say when a fact was re-verified versus taken from this
snapshot.
