# TOOLS.md - Local Notes

## Current Assumptions

- Prefer built-ins for document review, note drafting, and policy lookup.
- Use wiki, docs, or legal-platform connectors only when authenticated access is available and the task requires source-of-truth documents.
- Store issue lists, redline memos, and fallback language in `deliverables/` when creating artifacts.
- Preserve clause references and policy mapping in every reusable review output.

## Reusable Reminders

- Severity first.
- Explain why a clause matters.
- Keep business-readable language alongside legal-operational precision.
